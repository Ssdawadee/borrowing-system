# การยืนยันตัวตน (Authentication)

เอกสารนี้อธิบายกลไกการเข้าสู่ระบบและการควบคุมสิทธิ์ของระบบยืม-คืนอุปกรณ์

## บทบาทผู้ใช้ในระบบ

- `admin` ผู้ดูแลระบบ
- `user` นักศึกษา

## เส้นทางที่เกี่ยวข้อง

- `POST /api/auth/register` (สมัครสมาชิก)
- `POST /api/auth/login` (เข้าสู่ระบบ)

## สมัครสมาชิก (Register)

ใช้สำหรับนักศึกษาสมัครสมาชิกใหม่ (admin เพิ่มผู้ใช้ไม่ได้)

**Field ที่ต้องใช้:**
- `student_id` (ขึ้นต้น b + เลข 10 หลัก)
- `name` (ไม่ว่าง)
- `phone` (ตัวเลข 10 หลัก)
- `password` (ไม่ว่าง)
- `email` (optional, ถ้าไม่ระบุจะสร้างอีเมลอัตโนมัติ)

เมื่อสมัครสมาชิกสำเร็จ ระบบจะสร้างบัญชีประเภท `user` เท่านั้น

## เข้าสู่ระบบ (Login)

**Field ที่ต้องใช้:**
- `identifier` (student_id หรือ email)
- `password`
- `role` (user หรือ admin)

ถ้าเลือก role ไม่ตรงกับบัญชี จะไม่สามารถเข้าสู่ระบบได้

## การใช้ Token

API ที่ต้องยืนยันตัวตน ต้องส่ง header:

```
Authorization: Bearer <token>
```

## Middleware

- `authMiddleware` ตรวจสอบ JWT token
- `roleMiddleware` ตรวจสอบสิทธิ์ role (เช่น admin)

## หมายเหตุ

- บัญชีที่ถูกปิดการใช้งาน (`is_active = 0`) จะไม่สามารถเข้าสู่ระบบได้
- route ที่ไม่เกี่ยวข้องกับผู้ดูแลจะไม่สามารถใช้บัญชี admin login ได้
