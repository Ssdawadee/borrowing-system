# บทบาทผู้ใช้ในระบบ (User Roles)

ระบบนี้มีบทบาทผู้ใช้ 2 ประเภทหลัก โดยแต่ละบทบาทมีสิทธิ์และข้อจำกัดตาม logic backend จริง

## 1) Admin (ผู้ดูแลระบบ)

**สิทธิ์หลัก**

- ดู dashboard ฝั่งผู้ดูแล
- อนุมัติ/ปฏิเสธคำขอยืม (เฉพาะสถานะ PENDING)
- ยืนยันการคืนอุปกรณ์ (เฉพาะสถานะ RETURN_PENDING)
- ดูประวัติการยืม-คืนทั้งหมด
- จัดการหมวดหมู่ (เพิ่ม/ลบ เฉพาะหมวดหมู่ที่ไม่มีอุปกรณ์ใช้งานอยู่)
- จัดการอุปกรณ์ (เพิ่ม/แก้ไข/ลบ)
- จัดการผู้ใช้ (ดูรายชื่อและลบได้เฉพาะบัญชี user ที่ไม่มีรายการยืมหรือคืนค้างอยู่)

**ข้อจำกัด**

- ไม่สามารถลบบัญชี admin ผ่านหน้า UI ได้
- ไม่สามารถลบหมวดหมู่ที่มีอุปกรณ์ใช้งานอยู่
- การอนุมัติ/ปฏิเสธ/ยืนยันคืน ต้องตรวจสอบสถานะล่าสุดของคำขอ

**เมนูที่เห็นในระบบ**

- แดชบอร์ด
- อนุมัติคำขอ
- ยืนยันการคืน
- ประวัติทั้งหมด
- จัดการหมวดหมู่
- จัดการอุปกรณ์
- จัดการผู้ใช้

## 2) User (นักศึกษา)

**สิทธิ์หลัก**

- ดูรายการอุปกรณ์
- ส่งคำขอยืมอุปกรณ์ (สูงสุด 3 ชิ้น/ชนิดต่อบัญชี)
- ดูประวัติการยืมของตนเอง
- ส่งคำขอคืนอุปกรณ์ (เฉพาะที่สถานะ APPROVED)
- ดู dashboard ฝั่งนักศึกษา

**ข้อจำกัด**

- ไม่สามารถเข้าถึง route หรือเมนู admin ได้
- ไม่สามารถยืมอุปกรณ์เกินจำนวนที่ระบบกำหนด
- ไม่สามารถคืนอุปกรณ์ที่ไม่ได้อยู่ในสถานะ APPROVED

**เมนูที่เห็นในระบบ**

- แดชบอร์ด
- รายการอุปกรณ์
- ประวัติการยืม
- คืนอุปกรณ์

## หมายเหตุ

- route ที่เป็นงานผู้ดูแลจะถูกป้องกันด้วย role `admin` (roleMiddleware)
- ผู้ใช้ role `user` ไม่สามารถเข้าหน้า/เรียก API ฝั่งผู้ดูแลได้ (authMiddleware + roleMiddleware)
- บัญชีที่ถูกปิดการใช้งานจะไม่สามารถ login ได้
